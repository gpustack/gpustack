# Deploying GLM-5.2-W4AFP8 with GPUStack and SGLang

This tutorial guides you through setting up **GLM-5.2-W4AFP8**—a quantized model with massive long-context capabilities—using **SGLang (v0.5.14)** in the **GPUStack** Custom Container mode. 

The configuration leverages advanced optimizations derived from your parameters in image_dbb9ee.png, including an 800k ultra-long context window, EAGLE speculative decoding, and NSA (Native Sparse Attention) prefill context parallelism for low-latency production environments.

---

## 🛠️ Prerequisites & Setup

### 1. Base Image
* **Image Name**: `lmsysorg/sglang:v0.5.14`
* **Hardware Requirements**: Given `--tp-size 8` and `--attn-cp-size 8`, an **8-GPU cluster interconnected with NVLink or RoCEv2** (e.g., 8x H20 or 8x 910B) is highly recommended.

### 2. Model Weights Source
* **Model Name**: `GLM-5.2-W4AFP8`
* **Download Source (ModelScope)**: `https://modelscope.cn/models/PhalaCloud/GLM-5.2-W4AFP8`

---

## 🚀 GPUStack Configuration Steps

Navigate to your GPUStack admin dashboard, go to **Models** -> **Add Model**, and select the **Custom Container** / **SGLang Backend** mode.

### 1. Basic Information
* **Model Name**: `GLM-5.2-W4AFP8`
* **Container Image**: `lmsysorg/sglang:v0.5.14`

### 2. Environment Variables
To fetch the weights at high speed from ModelScope, inject the following environment variables or use a setup script within the container:
