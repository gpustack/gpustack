# Deploying GLM-5.2-W4AFP8 with GPUStack and SGLang

This tutorial guides you through setting up **GLM-5.2-W4AFP8**—a quantized model with massive long-context capabilities—using **SGLang (v0.5.14)** in the **GPUStack** Custom Container mode. 

The configuration leverages advanced optimizations derived from your parameters, including an 800k ultra-long context window, EAGLE speculative decoding, and NSA (Native Sparse Attention) prefill context parallelism for low-latency production environments.

---

## 🛠️ Prerequisites & Setup

### 1. Base Image
* **Image Name**: `lmsysorg/sglang:v0.5.14`
* **Hardware Requirements**: H20

### 2. Model Weights Source
* **Model Name**: `GLM-5.2-W4AFP8`
* **Download Source (ModelScope)**: `https://modelscope.cn/models/PhalaCloud/GLM-5.2-W4AFP8`

---

## 🚀 GPUStack Configuration Steps

Navigate to your GPUStack admin dashboard, go to **Models** -> **Add Model**, and select the **Custom Container** / **SGLang Backend** mode.

### 1. Basic Information
* **Model Name**: `GLM-5.2-W4AFP8`
* **Container Image**: `lmsysorg/sglang:v0.5.14`

### 2. Backend Arguments (Multi-line Format for UI Input)
You can directly copy and paste the following parameters into the **Backend Arguments** text area in GPUStack (each argument occupies a single line without backslashes):

```text
--model-path /data/models/PhalaCloud/GLM-5.2-W4AFP8
--quantization w4afp8
--context-length 800000
--disable-shared-experts-fusion
--tp-size 8
--kv-cache-dtype fp8_e4m3
--reasoning-parser glm45
--tool-call-parser glm47
--mem-fraction-static 0.8
--trust-remote-code
--speculative-algorithm EAGLE
--speculative-num-steps 5
--speculative-eagle-topk 1
--speculative-num-draft-tokens 6
--cuda-graph-max-bs 32
--enable-nsa-prefill-context-parallel
--nsa-prefill-cp-mode round-robin-split
--attn-cp-size 8
