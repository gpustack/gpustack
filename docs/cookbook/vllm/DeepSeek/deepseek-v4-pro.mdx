# Deploying DeepSeek-V4-Pro with GPUStack and vLLM

This tutorial guides you through setting up **DeepSeek-V4-Pro**—a flagship Mixture-of-Experts (MoE) model featuring advanced optimization frameworks—using **vLLM (v0.24.0)** in the **GPUStack** Custom Container mode. 

---

## 🛠️ Prerequisites & Setup

### 1. Base Image
* **Image Name**: `vllm/vllm-openai:v0.24.0-cu129`
* **Hardware Requirements**: H20 (8x GPUs for Parallelism execution)

### 2. Model Weights Source
* **Model Name**: `DeepSeek-V4-Pro`
* **Download Source (ModelScope)**: `https://modelscope.cn/models/deepseek-ai/DeepSeek-V4-Pro`

---

## 🚀 GPUStack Configuration Steps

Navigate to your GPUStack admin dashboard, go to **Models** -> **deployment**, and select the **model deploy** / **modelscope** mode.

### 1. Basic Information
* **Model Name**: `DeepSeek-V4-Pro`
* **Container Image**: `vllm/vllm-openai:v0.24.0-cu129`

### 2. Backend Arguments (Multi-line Format for UI Input)
Based on the UI screenshot `image_de705f.png`, you can directly copy and paste the following parameters into the **server Arguments** text area in GPUStack (each argument occupies a single line without backslashes):

```text
--trust-remote-code
--kv-cache-dtype fp8
--block-size 256
--enable-expert-parallel
--data-parallel-size 8
--compilation-config '{"mode": 0, "cudagraph_mode": "FULL_DECODE_ONLY"}'
--gpu-memory-utilization 0.95
--max-model-len 500000
--tokenizer-mode deepseek_v4
--tool-call-parser deepseek_v4
--enable-auto-tool-choice
--reasoning-parser deepseek_v4
--max-num-seqs 512
--max-num-batched-tokens 512
--no-enable-flashinfer-autotune
--speculative_config '{"method":"mtp","num_speculative_tokens":2}'
--enable-prefix-caching
--default-chat-template-kwargs={"enable_thinking":false}
--performance-mode interactivity
```
