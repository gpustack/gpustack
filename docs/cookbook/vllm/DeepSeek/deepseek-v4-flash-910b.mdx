# Deploying DeepSeek-V4-Flash-Dspark with GPUStack and vLLM

This tutorial guides you through setting up **DeepSeek-V4-Flash-dspark**—a high-performance Mixture-of-Experts (MoE) model with massive long-context capabilities—using **vLLM-ascend (v0.25.1)** in the **GPUStack** Custom Container mode.

## 🚀 GPUStack Configuration Steps

Navigate to your GPUStack admin dashboard, go to **Models** -> **deployment**, and select the **model deploy** / **modelscope** mode.

### 1. Basic Information
* **Model Name**: `DeepSeek-V4-Flash-Dspark`
* **Container Image**: `swr.cn-north-4.myhuaweicloud.com/desaysv/vllm-ascend-dspark:910b4`

### 2. Backend Arguments (Multi-line Format for UI Input)
Based on the UI screenshot `image_de6826.png`, you can directly copy and paste the following parameters into the **server Arguments** text area in GPUStack (each argument occupies a single line without backslashes):

```text
--max-model-len 800000 
--max-num-batched-tokens 8192 
--served-model-name dsv4-dspark 
--gpu-memory-utilization 0.9
--max-num-seqs 32
--data-parallel-size 1
--tensor-parallel-size 8 
--enable-expert-parallel 
--tokenizer-mode deepseek_v4 
--tool-call-parser deepseek_v4 
--enable-auto-tool-choice 
--reasoning-parser deepseek_v4 
--no-disable-hybrid-kv-cache-manager 
--model-loader-extra-config='{"enable_multithread_load": true, "num_threads": 128}' 
--quantization ascend 
--block-size 128 
--speculative-config '{"method": "dspark", "num_speculative_tokens": 5, "enforce_eager": true}' 
```

env
```text
export OMP_NUM_THREADS=10
export PYTORCH_NPU_ALLOC_CONF=expandable_segments:True
export LD_PRELOAD=/usr/lib/aarch64-linux-gnu/libjemalloc.so.2:$LD_PRELOAD
export HCCL_BUFFSIZE=1024
export TASK_QUEUE_ENABLE=1
```
